提案日: 2026年4月1日 提案者: 井上 洋平 部門: 開発・自動化室
1

背景と現状の課題

AI_Company（AI仮想組織）はNotion・Asana・ローカルフォルダ上に構築済みですが、実際にAI部署を呼び出すには毎回Claudeの画面を開き、システムプロンプトを切り替える手間が発生しています。

課題 1
Slack ↔ Claude間の切り替えコストが高い
課題 2
部署ごとのプロンプト選択が煩雑
課題 3
AI活用が属人化しやすい
課題 4
利用状況が可視化できていない
2

提案概要

Slackのスラッシュコマンドで、日常業務のワークフロー内からAI部署に直接指示

例えば /email 米沢市観光課への御礼メール と入力するだけで、秘書室のAIが適切なビジネスメールを即座に生成します。チャットの流れを止めずに、専門AIの力を活用できます。

システム構成

💬
Slack
/command 入力
→
⚡
GAS Web App
コマンド判定
部署ルーティング
→
🧠
Claude API
部署別プロンプトで
AI応答生成
→
📨
Slack
結果を投稿

GAS（Google Apps Script）はGoogleの無料サーバー環境。
新規サーバー契約やインフラ管理は不要です。

セキュリティ対策

🔒
APIキーの安全管理
GASのPropertiesServiceで暗号化保存。コード内にハードコードしない
🛡️
リクエスト認証

レートリミット
